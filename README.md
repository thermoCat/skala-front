# SKALA Frontend

SKALA 프론트엔드 실습 과제 모음. HTML/CSS/JS만으로 만든 정적 사이트이며,
서버 없이 브라우저 `localStorage`로 로그인·개인화·데이터 저장을 구현했다.

## 실행 방법

`html/index.html`을 브라우저로 열면 된다. 별도 서버나 설치 과정이 없다.

## 폴더 구조

```
html/   페이지별 HTML (아래 목록 참고)
js/     공통 로직 (auth.js, alarms.js, timetable.js)
media/  여행 앨범 예시 사진 자리 (media/README.md 참고)
```

## 페이지 목록

| 페이지 | 설명 |
| --- | --- |
| `index.html` | 메인 Hub. 오늘의 일정, 여행 앨범 한 컷, 바로가기 메뉴. 비로그인 시 블러 처리 |
| `logIn.html` / `signUp.html` / `signUpResult.html` | 로그인 · 회원가입 |

| `myMemo.html` | 개인 메모장 |

## 핵심 기능

- **회원가입/로그인** — `js/auth.js`가 `localStorage`에 회원 정보와 로그인 세션을 저장.
  `data-personal`, `data-auth` 속성만 붙이면 어느 페이지든 로그인 상태에 따라
  내용이 자동으로 바뀐다.
- **개인화** — 소개 문구, 좋아하는 것/할 일/설명 단어(각 3칸 고정 입력), 여행 사진,
  휴일 계획, 메모가 전부 회원별로 따로 저장된다.
- **여행 사진 업로드** — 사진을 고르면 제목·내용 입력 창이 뜬 뒤 게시. 브라우저에서
  자동으로 축소 압축(긴 변 1600px, JPEG)해서 저장 용량을 아낀다.
- **오늘의 일정** — 강의 시간표(고정)와 그날의 휴일 계획을 시간순으로 합쳐서 표시.
- **알림** — 평일 08:50/17:50 입실·퇴실 체크와 휴일 계획 시각 알림을 브라우저
  알림(Notification API)으로 제공. 브라우저를 꺼둬도 울리도록 `.ics` 캘린더
  파일 내보내기(맥 캘린더/아웃룩/구글 캘린더 등록용)도 지원.

## 알아둘 점

- 서버가 없어 비밀번호도 `localStorage`에 그대로 저장된다 (실습용 한정).
- 브라우저 알림은 이 사이트 탭이 열려 있는 동안에만 동작한다. 컴퓨터가 꺼져
  있거나 브라우저가 완전히 종료된 상태에서 울리게 하려면 `.ics` 캘린더 등록을
  사용해야 한다.
- 데이터가 브라우저별로 저장되므로, 다른 브라우저·시크릿 모드·다른 컴퓨터에서는
  가입 정보와 게시물이 보이지 않는다.

## 다른 컴퓨터에서 이어서 작업하기

```
git clone https://github.com/thermoCat/skala-front.git
```

- **`html/myClass.html`은 절대 수정하지 않는다** — 시간표 고정 과제라 처음부터
  손대지 않기로 한 파일.
- 커밋은 한국어 컨벤셔널 커밋(`feat:`, `fix:`, `docs:`)으로 작성하고,
  `Co-Authored-By: Claude` 등 AI 관련 표기는 넣지 않는다.
- 회원·사진·계획·메모는 전부 브라우저 `localStorage`에만 저장된다. 컴퓨터를
  옮기면 이전에 만든 테스트 계정과 데이터는 남아있지 않다 (정상 동작이며
  버그가 아니다).
- **미완/확인 필요:** `myTrip.html`은 사용자 요청으로 배경음악(`<audio>`)과
  브이로그(`<video>`)를 제거한 상태다. 원래 과제 스펙은 이 두 요소를 필수로
  요구하므로, 제출 전 다시 넣을지 확인이 필요하다.
- 알림 기능은 실제 기기에서 각 페이지의 "🧪 지금 테스트 알림 보내기" 버튼으로
  직접 켜고 확인해야 한다 (브라우저 알림 권한은 기기·브라우저별로 따로 설정됨).
